Integer Out of Bound

Problems where overflow is a hidden trap or the core constraint
Java/C++ limits: int → ±2,147,483,647 (2³¹−1)  |  long → ±9,223,372,036,854,775,807 (2⁶³−1)
Common traps: multiplication before division · mid = (lo+hi)/2 · factorial/combination · reversing digits · recursive sum accumulation
Pattern 1 Digit / bit reversal overflow O(log n)
Reversing an integer — result may exceed 32-bit range

Pattern 2 Binary search mid overflow O(log n)
Classic (lo + hi) / 2 overflows when both are near INT_MAX
No.ProblemOverflow trapDiff
69 Sqrt(x) mid*mid overflows int; use long Easy
29 Divide Two Integers INT_MIN / -1 = INT_MAX + 1 Med
50 Pow(x, n) n = INT_MIN → -n overflows; use long n Med
Fix: mid = lo + (hi − lo) / 2 — never (lo + hi) / 2. For products use long.
Pattern 3 Modular arithmetic (large numbers) O(n) or O(log n)
Answer is huge — return result modulo 10⁹+7 or similar prime

Quick decision guide
Reversing / parsing digits?→Pattern 1 — check before multiply
Binary search or exponentiation?→Pattern 2 — lo + (hi−lo)/2, cast to long
Answer mod 10⁹+7?→Pattern 3 — mod every op, use fast power
Numbers too large for any int type?→Pattern 4 — string arithmetic / BigInteger
